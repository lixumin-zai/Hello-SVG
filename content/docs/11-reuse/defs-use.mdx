## 11.1.1 `<defs>`：定义资源

`<defs>` 标签用于包裹那些**你不希望直接显示出来**的内容。它就像是一个仓库。

```xml
<svg>
  <defs>
    <!-- 这里面的东西不会画在屏幕上 -->
    <circle id="myCircle" r="10" fill="red" />
  </defs>
  
  <!-- 这里是空的，直到你引用它 -->
</svg>
```

## 11.1.2 `<use>`：克隆实例

`<use>` 标签用于取出 `<defs>` 中的资源，并把它画在画布上。

```xml
<use href="#myCircle" x="20" y="20" />
<use href="#myCircle" x="50" y="50" />
```

-   **`href`**：指向资源的 ID。
-   **`x`, `y`**：实例的偏移量。

### Shadow DOM (影子 DOM)

当你使用 `<use>` 时，浏览器实际上创建了一个**影子 DOM**。这意味着：
1.  你无法用 CSS 直接选中 `<use>` 内部的子元素（例如 `use circle { fill: blue }` 是无效的）。
2.  样式继承：`<use>` 元素本身就像一个容器。如果你给 `<use>` 设置 `fill="blue"`，而原始图形**没有设置 fill**（或者设为 `currentColor`），那么它会继承蓝色。如果原始图形已经写死了 `fill="red"`，那么 `<use>` 上的样式无法覆盖它。

这就是为什么在制作图标系统时，我们通常会把图标的 `fill` 删掉，或者设为 `currentColor`。
