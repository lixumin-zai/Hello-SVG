在 Web 开发中，如果我们想知道一个 HTML `<div>` 的宽高，我们可以使用 `getBoundingClientRect()`。
对于 SVG 元素，浏览器不仅提供了类似的包围盒计算，还提供了一套极其强大的**几何计算 API**，尤其是针对复杂曲线 `<path>` 的计算接口。

## 交互探索：路径长度与坐标定位

如果你有一条复杂的贝塞尔曲线，你想让一个物体（比如小汽车）沿着这条曲线精确地开过去，或者你想做一条逐渐被画出来的线，你需要知道什么？
1. 这条线**到底有多长**？（通过 `getTotalLength()`）
2. 在进度为 50% 时，它的 **X, Y 坐标**在哪里？（通过 `getPointAtLength()`）

## 紧凑包围盒 (BBox)

通过 `getBBox()` 可以获取 SVG 元素的“紧凑包围盒”（Bounding Box）。它返回的对象包含 `x`, `y`, `width`, `height`。

这与 HTML 的 `getBoundingClientRect()` 不同：
- `getBBox()` 返回的是元素在其**本地用户坐标系**中的尺寸，不受 CSS `transform` 缩放/旋转的影响。
- **它只计算图形的几何边界，不包含描边 (stroke) 的宽度！**

## 13.3 坐标转换实战 (Screen CTM)

在处理 SVG 交互（例如拖拽、自定义绘图板）时，我们经常需要把屏幕坐标（鼠标事件的 `clientX` / `clientY`）转换为 SVG 内部的用户坐标系。因为 SVG 画布可能被缩放了（通过 `viewBox`），甚至可能被 CSS 变形了，所以两者的坐标往往不是 1:1 的。

### 使用 `getScreenCTM()` 转换坐标

现代浏览器的终极解法是利用 CTM（坐标变换矩阵）。在下方的区域中移动鼠标，观察屏幕 DOM 坐标是如何被转换成 SVG 内部绝对坐标的：

<Callout type="tip" title="为什么要用这个公式？">
  无论你的 SVG 页面如何滚动、无论设置了什么奇葩的 `viewBox`，无论外部被 CSS 怎样缩放，利用 `svg.getScreenCTM().inverse()` 这套计算矩阵都能保证你获取到鼠标在 SVG 世界中的绝对精确位置！
