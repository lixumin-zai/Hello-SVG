在 HTML 中，元素是一个个矩形盒子，鼠标只要进入盒子的矩形区域，就会触发交互。
但在 SVG 中，元素是各种不规则的几何图形。你画了一个空心圆（只有描边，没有填充），当你把鼠标放在圆心（空心区域）时，算不算“悬停在这个圆上”？

这就引出了 SVG 交互中最核心的属性：**`pointer-events`**。

## `pointer-events` 核心值详解

`pointer-events` 最初是 SVG 专属属性（后来才被引入 CSS 中用于 HTML）。在 SVG 中，它的可选值远比在 HTML 中丰富：

*   **`visiblePainted` (默认值)**：元素必须可见（`visibility` 不是 hidden）。鼠标必须落在元素的 **填充区(fill)** 或 **描边区(stroke)** 才会触发。如果 `fill="none"`，点击圆心是**不会**触发事件的！
*   **`visibleFill`**：只要鼠标在填充区内就触发，不管你设没设置 `fill`（即使是 `fill="none"` 也触发！）。
*   **`visibleStroke`**：只要鼠标在描边区上就触发，不管你设没设置 `stroke`（即使是 `stroke="none"` 也触发！）。
*   **`all`**：不管元素是否可见（哪怕 `visibility="hidden"`），也不管有没有填充和描边，只要鼠标落在它的几何范围内就触发。这经常用来做**不可见的“点击热区” (Hitbox)**。
*   **`none`**：元素彻底变成“幽灵”，鼠标会穿透它，点击到底层的元素上。

<Callout type="tip" title="实战技巧：隐形的热区">
  在做交互式图表时，如果数据点（小圆圈）太小，用户很难用鼠标精准点中。最佳做法是：在小圆点外面再画一个很大的透明圆（`fill="transparent" pointer-events="all"`），把事件绑定在这个大圆上！
</Callout>

## 事件冒泡与事件委托 (Event Delegation)

SVG 的事件流（捕获和冒泡）与普通的 HTML DOM 完全一致。
正因为冒泡机制的存在，当我们有一个包含 1000 个多边形的中国地图时，**千万不要给每个多边形绑定事件！**

正确的做法是利用**事件委托**：把唯一的事件监听器绑定在父级 `<svg>` 或 `<g>` 上，然后通过 `event.target` 判断用户点击的是哪个具体的子元素。
