# 6.4 坐标系变换高级话题

在复杂的 SVG 场景中（如游戏、数据可视化图表），我们通常不会把所有变换都塞给一个元素。相反，我们会利用分组 `<g>` 来建立层级化的坐标系。

## 6.4.1 嵌套变换 (Nested Transforms)

子元素的变换是建立在父元素变换的基础之上的。

```xml
<g transform="translate(100, 100)">
  <!-- 这个组建立了一个原点在 (100, 100) 的新世界 -->
  
  <g transform="rotate(45)">
    <!-- 这个组在 (100, 100) 的基础上旋转了 45 度 -->
    
    <rect x="0" y="0" ... />
    <!-- 这个矩形画在旋转后的世界里 -->
  </g>
</g>
```

这种层级结构使得“局部坐标系”成为可能。例如，画一辆车：
1.  车身组：负责整体移动 `translate(x, y)`。
2.  轮子组（在车身组内）：负责自转 `rotate(angle)`。
这样，当车移动时，轮子会自动跟着走，我们只需要关心轮子的自转即可。

## 6.4.2 笛卡尔坐标系翻转

SVG 的 Y 轴默认向下，这与数学中的笛卡尔坐标系（Y 轴向上）相反。
如果你在做数据可视化，可能希望 Y 轴向上。

技巧：使用 `scale(1, -1)` 翻转 Y 轴。

```xml
<g transform="translate(0, 400) scale(1, -1)">
  <!-- 这里的 (0, 0) 现在在画布底部 -->
  <!-- y=100 会向上画，而不是向下 -->
  <rect ... />
</g>
```
**副作用**：所有的文字也会被倒过来！你需要对文字再次应用 `scale(1, -1)` 把它翻回去。

## 6.4.3 总结

变换是 SVG 的灵魂。它赋予了静态图形运动的能力，也赋予了我们构建复杂层级结构的能力。

掌握了：
1.  `transform` 属性的基本操作。
2.  变换顺序的影响。
3.  利用 `<g>` 进行层级管理。

你就拥有了构建复杂 SVG 应用的钥匙。
