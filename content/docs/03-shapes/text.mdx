SVG 文本（`<text>`）与 HTML 文本有着根本的区别：**它是一种图形**。

## 3.5.1 `<text>` 元素：定位与基线

```xml
<text x="50" y="50" font-size="20">Hello World</text>
```

- **`x`, `y`**：文本起点的坐标。
- **基线对齐**：默认情况下，`y` 坐标指的是**基线（Baseline）**的位置，而不是文本的顶部或底部。这意味着如果你设 `y="0"`，你会发现字被切掉了一半（只露出基线以下的部分，如 g, j, p, q, y 的尾巴）。

## 3.5.2 `<tspan>`：行内文本分组

SVG 不支持自动换行（word-wrap）。如果你想分行显示，必须手动拆分 `<text>` 或使用 `<tspan>`。

```xml
<text x="10" y="20">
  <tspan x="10" dy="1.2em">第一行</tspan>
  <tspan x="10" dy="1.2em">第二行</tspan>
</text>
```

- **`dy`**：相对于**前一个字符**位置的垂直偏移量。这对于实现多行文本非常有用。
- **`dx`**：水平偏移量。

## 3.5.3 `<textPath>`：文字沿路径排列

這是 SVG 的独门绝技：让文字沿着任意曲线流动。

```xml
<path id="curve" d="M10,80 C40,10 65,10 95,80" fill="none" />
<text>
  <textPath href="#curve">
    沿着波浪线流动的文字~
  </textPath>
</text>
```

- **`href`**：引用一个 `<path>` 元素的 ID。
- **`startOffset`**：文字在路径上的起始位置（可以是百分比）。

## 3.5.4 文本属性

- **`font-family`**：字体族。
- **`font-size`**：字号。
- **`font-weight`**：粗细。
- **`text-anchor`**：水平对齐方式。
  - `start`（默认）：左对齐。
  - `middle`：居中对齐。
  - `end`：右对齐。
- **`dominant-baseline`**：垂直对齐方式。
  - `auto`（默认）：基线对齐。
  - `middle`：垂直居中。
  - `hanging`：顶部对齐（常用于东亚文字）。

## 3.5.5 多行文本的挑战

SVG 最大的痛点就是**不支持自动换行**。
如果你需要像 HTML 那样显示大段文本，你有两个选择：
1.  **`<foreignObject>`**：嵌入 HTML `<div>`（最推荐）。
2.  **JS 计算**：手动测量每个字的宽度，用 `<tspan>` 换行（非常繁琐，但在导出为静态图片时必须这么做）。
