## 为什么不直接用 `<path>`？

你可能会问：*既然 `<path>`（路径）可以画出所有形状，为什么还需要 `<rect>` 或 `<circle>` 呢？*

1.  **语义化**：`<circle>` 一眼就能看出是圆，而 `<path d="M..." />` 需要读代码才能猜出来。
2.  **可读性**：参数简单直观（如 `r="50"` vs. 一堆贝塞尔曲线坐标）。
3.  **动画便利**：想要改变圆的半径，只需动画 `r` 属性；如果用 path，需要重新计算整个 `d` 属性。
4.  **文件体积**：基础形状通常比等效的 path 代码更短。

当然，如果你需要画一个不规则的形状，`<path>` 是你唯一的选择（我们将作为单独一章详细讲解）。
