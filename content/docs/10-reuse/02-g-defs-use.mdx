## 基础打包：`<g>` (Group)
`<g>` 标签是 SVG 中的 `<div>`。它本身没有视觉表现，但它有两个巨大的作用：
1. **统一变换**：你可以给 `<g>` 加一个 `transform`，里面的所有形状都会跟着移动或旋转。
2. **属性继承**：你可以给 `<g>` 设置 `fill="blue"`，里面所有的图形默认都会变成蓝色（除非某个图形自己硬编码了颜色）。

```xml
<g transform="translate(50, 50)" fill="blue">
  <circle cx="0" cy="0" r="20" />
  <rect x="20" y="20" width="40" height="40" />
</g>
```

## 声明变量：`<defs>` 与 `<use>`
当你画好了一辆复杂的小汽车，你想在画布上放 5 辆，难道要把代码复制 5 遍吗？
不用！把汽车放进 `<defs>`（Definitions 的缩写）里：

```xml
<defs>
  <g id="car">
    <!-- 复杂的汽车绘制代码 -->
    <rect x="0" y="20" width="100" height="40" fill="red" />
    <circle cx="20" cy="60" r="15" fill="black" />
    <circle cx="80" cy="60" r="15" fill="black" />
  </g>
</defs>
```

**`<defs>` 里的东西是不会被直接渲染出来的！** 它们就像是在内存中声明的变量。
当你想用它时，使用 `<use>` 标签（就像调用函数）：

```xml
<use href="#car" x="10" y="20" />
<use href="#car" x="100" y="100" />
```

<Callout type="warn" title="use 的限制">
  <code>&lt;use&gt;</code> 生成的是 Shadow DOM，你无法用 CSS 或 JS 深入进去修改汽车轮胎的颜色。
  它最大的痛点是：**你无法改变原图形的宽高比例！** 哪怕你在 <code>&lt;use&gt;</code> 上写了 <code>width="100"</code>，原图形也只会被生硬地裁剪，而不是等比缩放。
</Callout>
