# 1.1 SVG 的起源与发展史

SVG (Scalable Vector Graphics) 并非横空出世，它是图形技术数十年演进的结晶。要真正理解 SVG，我们需要回到它的起源。

## 1.1.1 矢量图形的历史背景

早在 Web 诞生之前，矢量图形就已经存在。
- **PostScript (1982)**: Adobe 推出的页面描述语言，奠定了现代数字出版的基础。它不仅是打印机的语言，也是 PDF 的前身。
- **EPS (Encapsulated PostScript)**: 基于 PostScript 的图形文件格式，长期作为印刷行业矢量图的标准。

然而，这些格式主要针对打印出版，体积庞大且不适合网络传输。随着 90 年代 Web 的兴起，人们迫切需要一种轻量级、基于文本的矢量图形格式。

## 1.1.2 W3C 的 SVG 标准化历程

1998 年，W3C 收到了六份关于 Web 矢量图形的提案，其中最重要的是：
- **VML (Vector Markup Language)**: 由 Microsoft、Macromedia 等提交（后来在 IE 中使用了很长时间）。
- **PGML (Precision Graphics Markup Language)**: 由 Adobe、IBM、Netscape 等提交（基于 PostScript 模型）。

W3C 并没有选择其中之一，而是成立了 SVG 工作组，吸收了 VML 和 PGML 的精华，于 **1999 年** 发布了 SVG 1.0 的第一份草案。

## 1.1.3 SVG 各版本演进

- **SVG 1.0 (2001)**: W3C 推荐标准。定义了基本的形状、路径、文本、变换、动画（SMIL）等。
- **SVG 1.1 (2003)**: 最广泛支持的版本。主要将 SVG 1.0 模块化，方便在不同设备（如手机）上实现子集（SVG Tiny, SVG Basic）。
- **SVG 1.2 (废弃)**: 曾尝试增加许多新特性（如 DOM 访问、渲染顺序控制），但因过于复杂且缺乏浏览器支持而流产。
- **SVG 2.0 (进行中)**: 目前的开发重点。旨在与 HTML5 和 CSS3 更好地集成，简化语法，移除不常用的特性（如部分 SMIL），并采纳 Web 开发者的实际需求。

## 1.1.4 浏览器支持历史与现状

SVG 的普及并非一帆风顺，它曾经历过漫长的“黑暗时期”。

- **IE6 - IE8**: 原生不支持 SVG！开发者不得不依赖 Adobe SVG Viewer 插件，或者使用 VML 作为降级方案（著名的 Raphaël.js 库就是为此而生）。
- **Chrome / Firefox / Safari**: 较早开始支持 SVG，但早期版本在性能和完整性上存在差异。
- **IE9+**: Microsoft 终于开始原生支持 SVG，标志着 SVG 可以在主流 Web 开发中使用了。
- **现代浏览器 (Evergreen Browsers)**: 如今，Chrome、Edge、Firefox、Safari 全面支持 SVG 1.1，并正在逐步实现 SVG 2.0 的特性。

## 1.1.5 SVG vs. Flash：一场时代之战

在 2000 年代，Web 动画和矢量图的霸主是 **Macromedia Flash** (后被 Adobe 收购)。

| 特性 | SVG | Flash (SWF) |
| :--- | :--- | :--- |
| **本质** | 开放标准 (W3C) | 私有格式 (Adobe) |
| **格式** | XML 文本 (可读、可索引) | 二进制 (不仅是图形，还有字节码) |
| **依赖** | 浏览器原生支持 | 需要安装 Flash Player 插件 |
| **SEO** | 搜索引擎友好 | 搜索引擎难以抓取 |
| **移动端** | iOS/Android 完美支持 | Steve Jobs 发表《Thoughts on Flash》后被移动端抛弃 |

最终，随着 HTML5 的崛起和移动互联网的爆发，**SVG 赢得了这场战争**。Flash 于 2020 年正式停止支持，而 SVG 成为了现代 Web 图形的基石。
