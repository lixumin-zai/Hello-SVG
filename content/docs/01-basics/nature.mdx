SVG 的核心在于“矢量”和“XML”。

## 1.2.1 什么是矢量图形？

计算机图形主要分为两类：**栅格（Raster）**和**矢量（Vector）**。

- **栅格图形（Raster Graphics）**：如 JPG, PNG, GIF。由像素网格组成，放大后会失真（马赛克）。
- **矢量图形（Vector Graphics）**：如 SVG, EPS, AI。由数学公式描述（点、线、曲线），无论放大多少倍，边缘永远清晰锐利。

- **左侧（Raster）**：随着放大倍数增加，你能清楚看到一个个方块（像素）。这是因为它记录的是每个点的颜色信息，放大就是把这些点变大。
- **右侧（Vector）**：无论放多大，圆依然是圆，文字依然清晰。这是因为它记录的是“画一个圆，半径20，中心在(25,25)”，每次放大都会重新计算绘制路径。

## 1.2.2 SVG 是 XML

SVG 全称 **Scalable Vector Graphics**，基于 **XML (Extensible Markup Language)**。这意味着 SVG 本质上是**文本文件**。

你可以用记事本打开任何 `.svg` 文件，你会看到类似这样的代码：

```xml
<svg width="100" height="100" viewBox="0 0 100 100">
  <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
</svg>
```

- **可读性**：你可以直接读懂图形结构。
- **可搜索性**：搜索引擎（Google）可以索引 SVG 中的文本内容。
- **可编程性**：可以用 JavaScript 操作 DOM 节点，用 CSS 修改样式。这是 Canvas 无法比拟的优势。

## 1.2.3 分辨率无关性：Retina 屏的救星

随着 Retina（视网膜）屏幕和 4K 显示器的普及，传统的栅格图片需要准备 `@1x`, `@2x`, `@3x` 等多套资源来保证清晰度。

SVG **一份文件，适配所有屏幕**。无论是在 320px 的手机上，还是在 5000px 的大屏幕上，它都保持完美清晰。这极大地简化了响应式设计的开发流程。

## 1.2.4 SVG 的使用场景全景图

SVG 不仅仅用于图标。在现代 Web 生态中，它扮演着多重角色：

- **图标系统（Icons）**：取代 Font Icons，支持多色、动画。
- **插画（Illustrations）**：扁平化风格插图，文件小，加载快。
- **数据可视化（Data Viz）**：D3.js 的基石，图表清晰且交互性强。
- **动画（Animation）**：路径动画、描边动画，实现复杂动效。
- **地图（Maps）**：交互式矢量地图，每个区域都是独立 DOM。
- **滤镜与特效（Filters）**：SVG Filter 可以应用于任何 HTML 元素（如毛玻璃、液态效果）。
