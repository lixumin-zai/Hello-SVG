SVG 并不是万能的。虽然它强大，但也有其适用边界。了解它与其他技术的优缺点，能帮助你做出更合适的技术选型。

## 1.3.1 SVG vs. Canvas

这是最常被问到的问题：**我该用 SVG 还是 Canvas？**

- **SVG (Scalable Vector Graphics)**：
    - **声明式（Declarative）**：你告诉浏览器“画一个圆”，浏览器帮你管理这个圆的状态。
    - **基于 DOM**：每个图形都是一个 DOM 元素。
    - **事件处理方便**：可以直接给圆绑定 `click`, `hover` 事件。
    - **性能瓶颈**：当图形数量非常多（如 > 10,000 个）时，DOM 操作会变慢。
    - **适用场景**：图表、地图、UI 界面、简单动画。

- **Canvas (HTML5 Canvas)**：
    - **命令式（Imperative）**：你需要一步步告诉浏览器“移动画笔到(x,y)，画线到(x2,y2)，填充红色”。
    - **位图（Bitmap）**：最终生成的是像素图，放大失真。
    - **无 DOM**：画完就忘，无法直接选中图形。事件处理需要自己计算坐标。
    - **高性能**：适合处理大量物体（如粒子系统），因为不涉及 DOM 操作。
    - **适用场景**：游戏、粒子特效、图像处理、复杂动画。

## 1.3.2 SVG vs. WebGL

- **WebGL (Web Graphics Library)**：
    - **基于 OpenGL ES**：利用 GPU 加速，性能极高。
    - **3D 渲染**：主要用于 3D 场景，虽然也可以做 2D，但开发成本高。
    - **适用场景**：大型 3D 游戏、复杂可视化、科学计算。

## 1.3.3 SVG vs. CSS Shapes & clip-path

CSS 现在也能画一些简单的图形（如圆角、渐变、多边形裁剪）。

- **CSS Shapes**：适合简单的装饰性图形（如三角形箭头）。
- **CSS clip-path**：可以裁剪图片为特定形状，但路径定义不如 SVG 灵活。
- **SVG**：适合复杂的图形组合、路径运算和精确控制。

## 1.3.4 SVG vs. PNG/JPEG/WebP

这不仅是矢量 vs. 栅格的问题，还涉及到文件格式的选择。

- **SVG**：适合线条清晰、颜色简单、需要缩放的图形（Logo、图标）。
- **PNG/JPEG**：适合照片、复杂纹理、色彩丰富的图像。
- **WebP**：兼顾两者优点，但本质上还是栅格图。

## 1.3.5 SVG vs. PDF

PDF 也是矢量格式，但它是为**打印**而生的。
- **PDF**：页面固定，难以响应式布局，交互性弱。
- **SVG**：天生适应 Web，响应式，交互性强。
它们有共同的祖先（PostScript），但在 Web 上，SVG 是绝对的王者。
