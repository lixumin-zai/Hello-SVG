## 栅格 vs. 矢量的根本区别

要理解 SVG，首先要弄清**光栅图形（Raster）**和**矢量图形（Vector）**的本质差异。

### 光栅图形 (Raster Graphics)

光栅图（也称位图）是由一个一个彩色的方形网格（像素）组成的图像。常见的格式有 JPEG、PNG、WebP、GIF。

*   **工作原理**：记录每个像素的颜色信息（例如：第 1 行第 1 列是红色，第 1 行第 2 列是深红色…）。
*   **优点**：非常适合表现色彩丰富、层次细腻的照片级真实图像，这是由于像素点能够记录丰富的色彩渐变和细节。
*   **缺点**：放大后会"失真"（出现马赛克锯齿）；图像尺寸越大，需要记录的像素越多，文件体积呈指数级增长。

### 矢量图形 (Vector Graphics)

矢量图是使用数学方程和几何图元（如点、线、多边形）来描述图像的。

*   **工作原理**：记录图形的绘制指令（例如：以坐标 `(10, 10)` 为圆心，画一个半径为 `5`，填充颜色为红色的圆）。
*   **优点**：无论放大多少倍，边缘永远平滑锐利；对于简单图形，只包含几何坐标和颜色信息，文件体积非常小。
*   **缺点**：不适合用来表现复杂的照片级真实图像（计算成本过高，且难以用简单几何体描述）。

## SVG 是 XML —— 文本、可读、可搜索

与由二进制数据构成的 JPEG 或 PNG 不同，**SVG 本质上是一个 XML 文本文件**。这也是为什么我们在网页上甚至可以直接通过 `<svg>` 标签将图形代码直接写在 HTML 里（即内联 SVG）。

这意味着 SVG 拥有几个无与伦比的优势：

1.  **人类可读**：你可以用任何文本编辑器打开它，并看懂里面的代码。
2.  **可被搜索引擎索引**：由于里面包含文本节点 `<text>`，搜索引擎可以轻松读取其中的文字信息，有利于 SEO。
3.  **支持压缩**：作为纯文本，SVG 非常适合 GZIP/Brotli 等文本压缩算法，压缩率极高，能极大减少网络传输时间。
4.  **可脚本化与样式化**：SVG 拥有与 HTML 相似的 DOM 结构，因此可以使用 JavaScript 动态操作它，也可以使用 CSS 为其添加样式、触发动画。

## 分辨率无关性：Retina 屏的救星

在 Retina 和高分屏（如 4K 甚至 8K 显示器）普及的今天，前端开发者经常要为了一张位图准备 `@1x`、`@2x`、`@3x` 多个版本，不仅增加了工作量，也显著增加了应用体积。

SVG 彻底解决了这个问题：**一张图，适配所有分辨率和屏幕尺寸**。无论屏幕像素密度多高，浏览器底层绘图引擎总会以当前屏幕的最佳分辨率重新渲染 SVG，永远保持极致的清晰度。
