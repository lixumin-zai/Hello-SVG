在第十三部分，我们学习了如何使用原生 JavaScript DOM API (如 `createElementNS`、`setAttribute`) 来操作 SVG。那是最底层的原理，但在实际的企业级开发中，我们几乎不会那么做。

在现代前端开发中，我们通常使用 **React**、**Vue**、**Svelte** 或 **Angular** 等框架。这些框架的核心理念是**声明式渲染**和**组件化**，这与 SVG 的 XML 结构天然契合！

## 为什么要在框架中写 SVG？

1. **数据驱动图形**：不需要手动写繁琐的 DOM 操作代码。只要改变状态 (State/Ref)，框架会自动帮你更新 SVG 的 `x`、`y` 或 `fill` 属性。
2. **完美组件化**：一个复杂的图标、一个图表坐标轴，都可以被封装成独立的 `<MyAxis />` 组件，随处复用。
3. **Props 注入**：你可以通过传递 Props 来动态改变 SVG 的颜色、大小、动画状态。
