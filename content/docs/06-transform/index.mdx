如果你熟悉 CSS 的 `transform`，你可能会觉得 SVG 的变换很简单。**千万不要掉以轻心！** 
SVG 的变换系统虽然语法看起来一样，但底层逻辑（特别是坐标系和中心点）与 HTML/CSS 截然不同。无数前端开发者在这里踩过深坑。

## 核心难点与局部坐标系

这是你必须刻在脑子里的 SVG 铁律：**SVG 的所有变换（旋转、缩放、倾斜），默认永远是以画布的绝对原点 `(0,0)` 为中心进行的！** 
并且，`transform` 改变的并不是图形本身，而是改变了绘制这个图形的**局部坐标系**。

<TransformConceptsExplorer />

<Callout type="tip" title="CSS transform-origin 的兼容性">
  在现代浏览器中（SVG 2.0 规范），你可以直接使用 CSS 的 <code>transform-origin: center; transform-box: fill-box;</code> 来让 SVG 元素像 HTML 元素一样以自身中心点旋转缩放。
  <br/>但如果你需要兼容较老的浏览器，或者生成要导出给设计软件使用的纯 SVG 代码，最好还是老老实实计算并写死 <code>(cx, cy)</code> 坐标（比如 `rotate(45 cx cy)`）。
</Callout>
