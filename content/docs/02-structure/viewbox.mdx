SVG 文件的根元素是 `<svg>`。它就像 HTML 中的 `<html>` 标签一样，定义了文档的边界和规则。

## 2.1.1 两个世界：视口 vs. 视图框

在 SVG 中，有两个非常重要的矩形概念：

1.  **视口 (Viewport)**：由 `width` 和 `height` 属性决定。它决定了 SVG 在网页上**占据多大的屏幕空间**。就像你买的相框的大小。
2.  **视图框 (ViewBox)**：由 `viewBox` 属性决定。它决定了 SVG **内部世界的坐标系**。就像相框里照片拍摄的范围。

### 实验观察指南

1.  **平移 (Pan)**：增加 `min-x`，你会发现圆圈向左移动了。
    *   *原理*：你把相机的取景框向右移了，所以物体看起来向左跑了。
2.  **缩放 (Zoom)**：增加 `width`（viewBox 的 width，不是 svg 的 width），圆圈变小了。
    *   *原理*：你的相机视野变宽了（广角），所以物体看起来变小了。
3.  **变形 (Distortion)**：如果 viewBox 的宽高比与外层容器不同，`preserveAspectRatio` 就起作用了。
    *   试着把 `preserveAspectRatio` 设为 `none`，你会看到圆变成了椭圆。

## 2.1.2 `<svg>` 根元素详解

一个标准的 SVG 根元素通常包含以下属性：

```xml
<svg 
  width="100%" 
  height="100%" 
  viewBox="0 0 100 100"
  xmlns="http://www.w3.org/2000/svg"
>
  <!-- 内容 -->
</svg>
```

- **`width` / `height`**：如果不写单位，默认为像素 (px)。建议设为 `100%` 或具体的像素值。如果不写，默认是 `300x150`（和 iframe 一样）。
- **`viewBox="min-x min-y width height"`**：
  - `min-x`, `min-y`：左上角的坐标。
  - `width`, `height`：坐标系的宽和高。
- **`xmlns`**：命名空间（下一节详解）。

## 2.1.3 `preserveAspectRatio` 全解析

当 `viewBox` 的比例与 `width/height` 的比例不一致时，浏览器该怎么办？这就是 `preserveAspectRatio` 的职责。

它的语法是：`preserveAspectRatio="<align> <meetOrSlice>"`

1.  **`align` (对齐方式)**：
    - `xMidYMid`（默认）：在 X 轴居中，Y 轴居中。
    - `xMinYMin`：左上角对齐。
    - `xMaxYMax`：右下角对齐。
    
2.  **`meetOrSlice` (缩放策略)**：
    - `meet`（默认）：**包含**。保持比例，缩放到完全塞进视口，可能会留白边。
    - `slice`：**覆盖**。保持比例，缩放到填满视口，超出部分被裁剪（类似 CSS `object-fit: cover`）。
    - `none`：**拉伸**。不保持比例，强制拉伸填满（类似 CSS `object-fit: fill`）。

## 2.1.4 坐标系统

SVG 默认使用**笛卡尔坐标系**，但有一个重要区别：
- **原点 (0,0)**：在左上角。
- **X 轴**：向右为正。
- **Y 轴**：**向下为正**（这与数学课本不同！）。

这也是为什么在上面的实验中，增加 `min-y` 会让物体看起来向上移动（因为视野向下移动了）。
