如果你经常复制粘贴 SVG 代码，你一定见过 `xmlns` 和 `xlink`。它们到底是什么？为什么 HTML5 好像不需要它们？

## 2.3.1 SVG 命名空间

XML 命名空间（XML Namespace）是 XML 的核心机制，用于解决**同名标签冲突**的问题。
例如，XML 里可能有 `<table id="1">`（数据表），HTML 里也有 `<table>`（布局表）。浏览器怎么知道这是哪种表？

答案就是 **Namespace**。

### 标准 SVG 命名空间

```xml
<svg xmlns="http://www.w3.org/2000/svg">
  <!-- 内容 -->
</svg>
```

- **HTML5 时代**：如果你是直接在 `.html` 文件里写 SVG（内联），你可以**省略** `xmlns`。现代浏览器足够聪明，能猜到 `<svg>` 里面就是 SVG 内容。
- **独立 SVG 文件**：如果你保存为 `.svg` 文件（作为图片引用），**必须**加上 `xmlns`。否则浏览器会把它当成普通的 XML 文件显示（通常是一棵折叠树）。

## 2.3.2 XLink 命名空间（已废弃但常见）

在 SVG 1.1 中，引用资源（如 `<image href="...">` 或 `<use href="...">`）必须使用 XLink 命名空间。

```xml
<svg xmlns:xlink="http://www.w3.org/1999/xlink">
  <image xlink:href="logo.png" ... />
</svg>
```

- **SVG 2.0 变革**：现在你可以直接用 `href`，不需要 `xlink:` 前缀了！
  - **推荐写法**：`<image href="logo.png" />`
  - **兼容性**：所有现代浏览器都支持 `href`。为了兼容极老的浏览器（如 Safari 8），你可能还是会看到 `xlink:href`。

## 2.3.3 MathML 与 SVG 混用

SVG 可以嵌入其他 XML 格式，最常见的是 MathML（数学公式）。

```xml
<svg>
  <foreignObject width="100" height="50">
    <math xmlns="http://www.w3.org/1998/Math/MathML">
      <mi>x</mi>
      <mo>=</mo>
      <mfrac>
        <mrow>
          <mo>-</mo>
          <mi>b</mi>
          <mo>±</mo>
          <msqrt>
            <msup><mi>b</mi><mn>2</mn></msup>
            <mo>-</mo>
            <mn>4</mn><mi>a</mi><mi>c</mi>
          </msqrt>
        </mrow>
        <mrow><mn>2</mn><mi>a</mi></mrow>
      </mfrac>
    </math>
  </foreignObject>
</svg>
```

这利用了 `<foreignObject>` 元素，它允许你在 SVG 内部嵌入 HTML 或其他 XML 内容。这在制作复杂的数学图表时非常有用。

## 2.3.4 自定义数据属性 `data-*`

就像 HTML 一样，SVG 元素也支持 `data-*` 属性。这在 D3.js 等数据可视化库中极为常见，用于绑定数据。

```xml
<circle cx="50" cy="50" r="10" data-value="42" data-category="A" />
```

你可以通过 JS 的 `dataset` API 轻松访问这些数据：

```javascript
const circle = document.querySelector('circle');
console.log(circle.dataset.value); // "42"
```

## 总结

- **内联 SVG**：不需要 `xmlns`。
- **独立 .svg 文件**：必须要有 `xmlns="http://www.w3.org/2000/svg"`。
- **href 属性**：推荐直接用 `href`，不再需要 `xlink:href`。
