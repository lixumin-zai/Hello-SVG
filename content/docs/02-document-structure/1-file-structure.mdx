# 2.1 SVG 文件剖析

一个标准的 SVG 文件通常以 XML 声明开始，紧接着是 `<svg>` 根元素。理解这个根元素上的属性，是掌握 SVG 缩放特性的关键。

## 2.1.1 XML 声明与文档类型

虽然在现代 HTML5 中嵌入 SVG 时通常省略它们，但在独立的 `.svg` 文件中，你可能会看到：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">
<svg ...>
  ...
</svg>
```

- **XML 声明**: 告诉解析器这是一个 XML 文件。
- **DOCTYPE**: 历史遗留产物，在 SVG 2.0 中不再推荐使用。现在的最佳实践是直接写 `<svg>` 标签。

## 2.1.2 `<svg>` 根元素详解

`<svg>` 元素就像 HTML 中的 `<html>`，它是所有图形内容的容器。它最重要的三个属性是：

1.  **width / height**: 视口（Viewport）的尺寸。
2.  **viewBox**: 用户坐标系的定义。
3.  **preserveAspectRatio**: 当视口宽高比与 viewBox 宽高比不一致时的对齐策略。

### 视口 (Viewport) vs. 用户坐标系 (User Space)

这是一个让无数初学者晕头转向的概念。

- **视口 (Viewport)**: 就像是**窗户框**。由 `width` 和 `height` 属性（或 CSS）决定。它是 SVG 在网页上占据的实际像素区域。
- **viewBox**: 就像是**窗户外的风景**。它定义了 SVG 内部的坐标系。

### viewBox 属性

`viewBox` 属性的值由 4 个数字组成：`min-x`, `min-y`, `width`, `height`。

```xml
viewBox="0 0 100 100"
```

这意味着：SVG 内部建立了一个从 (0,0) 到 (100,100) 的坐标系。无论外面的 `<svg>` 标签被拉伸到多大（比如 1000px * 1000px），内部的一个单位永远等于视口的 1/100。**这就是 SVG "Scalable"（可缩放）的秘密！**

## 2.1.3 SVG 坐标系统

默认情况下，SVG 使用与屏幕相同的坐标系：
- **原点 (0,0)**: 左上角。
- **X 轴**: 向右增加。
- **Y 轴**: 向下增加。

这与数学中的笛卡尔坐标系（Y 轴向上）不同，需要适应。

## 2.1.4 嵌套 `<svg>` 与子坐标系

`<svg>` 标签是可以嵌套的！每个嵌套的 `<svg>` 都可以有自己的 `viewBox`，从而建立独立的局部坐标系。这在制作复杂的组合图形或图标系统时非常有用。

```xml
<svg viewBox="0 0 100 100">
  <!-- 主坐标系 -->
  <circle cx="50" cy="50" r="40" fill="blue" />
  
  <!-- 子坐标系：建立一个新的 20x20 的世界 -->
  <svg x="30" y="30" width="40" height="40" viewBox="0 0 20 20">
    <circle cx="10" cy="10" r="8" fill="red" />
  </svg>
</svg>
```

在这个例子中，红色的圆虽然半径只有 8，但在屏幕上看起来可能和蓝色圆（半径 40）差不多大，因为它们处于不同的坐标系缩放比例中。
