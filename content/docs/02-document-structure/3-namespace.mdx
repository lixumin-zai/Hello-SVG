# 2.3 SVG 命名空间与扩展

SVG 是基于 XML 的，而 XML 的一个核心特性就是**命名空间 (Namespace)**。虽然在 HTML5 中你可以不写 `xmlns`，但理解它对于处理 SVG 的高级特性（如链接、数学公式）至关重要。

## 2.3.1 SVG 命名空间 `http://www.w3.org/2000/svg`

任何 SVG 文件的根元素都必须声明其所属的命名空间：

```xml
<svg xmlns="http://www.w3.org/2000/svg">
  ...
</svg>
```

**为什么？**
浏览器看到 `<svg>` 标签时，需要知道它是 SVG 规范定义的那个 `<svg>`，而不是某个自创 XML 格式里的同名标签。

- **在 HTML5 中**: 现代浏览器足够聪明，如果你把 `<svg>` 直接写在 `.html` 文件里，解析器会自动识别并加上命名空间。
- **在独立 `.svg` 文件中**: **必须写！** 否则浏览器可能无法正确渲染，或者将其当作普通 XML 文本树处理。

## 2.3.2 XLink 命名空间 (历史遗留)

在 SVG 1.1 中，如果要引用外部资源（比如 `<use>`, `<image>`, `<a>`），你需要使用 `xlink:href` 属性。这就引入了另一个命名空间：

```xml
<svg xmlns="http://www.w3.org/2000/svg"
     xmlns:xlink="http://www.w3.org/1999/xlink">
  
  <image xlink:href="photo.jpg" width="100" height="100" />
  
  <use xlink:href="#my-icon" />
</svg>
```

**现状**:
SVG 2.0 规范已经废弃了 XLink。现在的最佳实践是直接使用 `href` 属性（不需要 `xlink:` 前缀）。

```xml
<!-- 现代写法 -->
<image href="photo.jpg" width="100" height="100" />
<use href="#my-icon" />
```

但是，为了兼容老旧浏览器（主要是 Safari < 12 和 iOS < 12），很多工具（如 SVGO）仍然会保留 `xlink:href`。

## 2.3.3 MathML 与 SVG 混用

XML 命名空间的真正威力在于混合不同格式的内容。你可以在 SVG 中嵌入 MathML（数学标记语言）来显示复杂的数学公式，反之亦然。

```xml
<svg viewBox="0 0 500 100" xmlns="http://www.w3.org/2000/svg">
  <foreignObject width="500" height="100">
    <!-- 这里切换到了 XHTML 命名空间 -->
    <div xmlns="http://www.w3.org/1999/xhtml">
      <!-- 再嵌入 MathML -->
      <math xmlns="http://www.w3.org/1998/Math/MathML">
        <mrow>
          <msup><mi>a</mi><mn>2</mn></msup>
          <mo>+</mo>
          <msup><mi>b</mi><mn>2</mn></msup>
          <mo>=</mo>
          <msup><mi>c</mi><mn>2</mn></msup>
        </mrow>
      </math>
    </div>
  </foreignObject>
</svg>
```

## 2.3.4 自定义数据属性 `data-*` 在 SVG 中的应用

就像在 HTML 中一样，你可以在 SVG 元素上使用 `data-*` 属性来存储自定义数据。

```xml
<circle cx="50" cy="50" r="10" 
        data-id="123" 
        data-category="user-node" 
        data-tooltip="点击查看详情" />
```

这对数据可视化非常有用。你可以用 D3.js 或原生 JS 读取这些属性：

```javascript
const circle = document.querySelector('circle');
console.log(circle.dataset.id); // "123"
```

这不仅符合规范，而且是实现交互式 SVG 图表（如地图区域点击、图表悬停提示）的标准做法。
