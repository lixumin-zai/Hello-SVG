# 2.2 SVG 嵌入方式大全

将 SVG 放到网页上有多种方法，每种方法都有其特定的优缺点。选择哪种方式取决于你是否需要**交互**、**样式控制**以及**性能**考虑。

## 2.2.1 内联 SVG (Inline SVG) —— 最强大的方式

这是目前最流行的方式，特别是当你需要用 CSS 或 JS 控制 SVG 时。

```html
<svg class="icon" viewBox="0 0 24 24" fill="currentColor">
  <path d="..." />
</svg>
```

- **优点**:
  - **CSS 样式**: 可以像普通的 DOM 元素一样设置样式（如 `fill`, `stroke`）。
  - **JS 交互**: 可以绑定事件监听器（`onclick`）。
  - **动画**: 可以使用 CSS 动画或 JS 动画库。
  - **HTTP 请求**: 不需要额外的 HTTP 请求（但也增加了 HTML 体积）。
- **缺点**:
  - 可能会使 HTML 变得臃肿。
  - 无法被浏览器缓存（因为它在 HTML 里面）。

## 2.2.2 `<img>` 标签 —— 最简单但限制最多

```html
<img src="icon.svg" alt="Icon" />
```

- **优点**:
  - **简单**: 和使用 PNG/JPG 一样。
  - **缓存**: 浏览器可以缓存 `.svg` 文件。
  - **语义化**: 对于装饰性图像，使用 `alt` 属性非常重要。
- **缺点**:
  - **无法控制样式**: 你不能用 CSS 修改 SVG 内部的颜色（除非使用 CSS Filter `filter: brightness(0) saturate(100%) ...` 这种黑魔法）。
  - **无法交互**: JS 无法访问 SVG 内部的 DOM。

## 2.2.3 CSS `background-image`

```css
.icon {
  background-image: url('icon.svg');
}
```

- **优点**:
  - **性能**: 可以使用 CSS Sprites 技术减少请求。
  - **语义**: 纯粹作为装饰，不影响 HTML 结构。
- **缺点**:
  - **无法控制样式**: 同样无法修改颜色。
  - **无障碍性**: 对屏幕阅读器不可见。

## 2.2.4 `<object>` 与 `<embed>` 标签

这是一种比较老派的方法，但在某些场景下仍然有用。

```html
<object data="image.svg" type="image/svg+xml">
  <!-- 降级显示 -->
  <img src="image.png" />
</object>
```

- **优点**:
  - **缓存**: 支持浏览器缓存。
  - **脚本执行**: SVG 内部的脚本可以执行（但在严格的 CSP 下可能受限）。
  - **交互**: 父页面可以通过 `contentDocument` 访问 SVG 内部 DOM（前提是同源）。
- **缺点**:
  - 写法繁琐。
  - 兼容性问题（尤其是 `<embed>`）。

## 2.2.5 `<iframe>` 嵌入

```html
<iframe src="image.svg"></iframe>
```

- **优点**:
  - **完全隔离**: SVG 在独立的文档环境中运行，样式互不干扰。
- **缺点**:
  - **重**: iframe 的开销比较大。
  - **难以交互**: 同样受同源策略限制，且操作繁琐。
  - **滚动条**: 如果尺寸不匹配，会出现滚动条。

## 2.2.6 Data URI (Base64 编码内联)

```css
.icon {
  background-image: url('data:image/svg+xml;base64,...');
}
```

- **优点**:
  - **减少请求**: 不需要额外的 HTTP 请求。
  - **便携**: 全部打包在一个 CSS 文件里。
- **缺点**:
  - **体积增大**: Base64 编码会比原始 SVG 大约增加 33% 的体积。
  - **无法缓存**: 除非整个 CSS 文件被缓存。
  - **难以维护**: 编码后的字符串不可读。

## 2.2.7 总结：能力对比表

| 方式 | CSS 样式控制 | JS 交互 | HTTP 请求 | 浏览器缓存 | 脚本执行 (Scripting) | CORS 限制 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: |
| **Inline SVG** | ✅ (强) | ✅ (强) | ❌ (0) | ❌ | ✅ | ❌ |
| **`<img>`** | ❌ | ❌ | ✅ | ✅ | ❌ (安全) | ✅ |
| **CSS Background** | ❌ | ❌ | ✅ | ✅ | ❌ | ✅ |
| **`<object>`** | ✅ (需跨文档) | ✅ (需跨文档) | ✅ | ✅ | ✅ | ✅ |
| **`<iframe>`** | ❌ | ❌ | ✅ | ✅ | ✅ | ✅ |
| **Data URI** | ❌ | ❌ | ❌ (0) | ❌ | ❌ | ❌ |

**最佳实践**:
- **图标系统**: 使用 **Inline SVG**（配合 React/Vue 组件）或 **SVG Sprites**。
- **插图/Logo**: 使用 **`<img>`**。
- **背景纹理**: 使用 **CSS Background**。
