## CSS 动画基础

在现代前端开发中，用 CSS 控制 SVG 动画是最符合组件化开发直觉的方式。你可以像控制普通 HTML 元素一样，使用 `transition` 和 `@keyframes` 控制 SVG 元素的颜色、透明度、位移等属性。

## 经典套路：画线动画 (Line Drawing)

"画线动画"是 SVG 中最经典的 CSS 动画效果，它利用了 `stroke-dasharray` (虚线间隙) 和 `stroke-dashoffset` (虚线偏移) 这对黄金搭档。

<Callout type="info" title="如何获取路径总长度？">
  如果你在写原生 JS，可以通过 `pathElement.getTotalLength()` 方法获取精确的路径长度。如果只是简单的动画，你也可以直接给 `stroke-dasharray` 赋一个足够大的近似值（比如 1000）。

## Transform Origin 陷阱 ⚠️

在 HTML 元素中，CSS 的 `transform-origin` 默认是元素的中心点（`50% 50%`）。但在 SVG 中，如果对内部的 `<rect>` 或 `<g>` 使用 CSS 旋转，你会遇到一个巨大的坑：

<TransformOriginVisualizer />

<Callout type="warn" title="总结与解法">
  - **问题**：SVG 中 CSS transform 的默认中心点是整个 SVG 画布的左上角 `(0, 0)`。如果你用 `50% 50%`，它指的是画布的中心，而不是元素自己的中心。
  - **终极解法**：在 CSS 中加上 `transform-box: fill-box;`。这样浏览器就会以元素自身的边界框（Bounding Box）来计算中心点！
</Callout>
