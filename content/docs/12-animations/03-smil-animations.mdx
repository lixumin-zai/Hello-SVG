## 什么是 SMIL？

SMIL（Synchronized Multimedia Integration Language）是 SVG 内置的动画标签系统。
它最大的优势是**不依赖外部样式和脚本**。如果你需要把 SVG 作为一张静态图片（例如用 `<img src="icon.svg">` 嵌入）并且还能动，那么 SMIL 是你唯一的选择！

<Callout type="warn" title="SMIL 的命运波折">
  Chrome 曾在 2015 年宣布要废弃 SMIL，这引发了社区的强烈抗议。后来因为 Web Animations API (WAAPI) 迟迟未能覆盖 SMIL 的所有功能（尤其是路径运动），Chrome 在 2016 年撤回了废弃决定。**现在你可以放心使用它了！**
</Callout>

## 核心元素介绍

### 1. `<animate>`：属性渐变
用于在两个属性值之间渐变（例如 `fill` 颜色、`r` 半径）。

```xml
<circle cx="50" cy="50" r="15" fill="blue">
  <!-- 改变颜色 -->
  <animate 
    attributeName="fill" 
    values="blue; red; blue" 
    dur="2s" 
    repeatCount="indefinite" 
  />
</circle>
```

### 2. `<animateTransform>`：变换动画
专门用于旋转、平移、缩放等变换。

```xml
<rect x="25" y="25" width="50" height="50">
  <!-- 围绕中心点 (50, 50) 旋转 -->
  <animateTransform 
    attributeName="transform" 
    type="rotate" 
    from="0 50 50" 
    to="360 50 50" 
    dur="3s" 
    repeatCount="indefinite" 
  />
</rect>
```

### 3. `<animateMotion>`：杀手级功能！
让元素沿着任意 `<path>` 路径运动，这是纯 CSS 很难做到的。

```xml
<svg viewBox="0 0 200 200">
  <!-- 运动轨迹 -->
  <path id="track" d="M 10 100 Q 100 10 190 100 T 190 190" fill="none" stroke="gray" />
  
  <circle r="10" fill="red">
    <animateMotion dur="4s" repeatCount="indefinite">
      <mpath href="#track" />
    </animateMotion>
  </circle>
</svg>
```
通过 `<mpath>`，动画元素可以完美地沿着定义好的轨迹路径循环运动。
