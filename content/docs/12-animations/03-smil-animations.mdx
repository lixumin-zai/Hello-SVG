## 什么是 SMIL？

SMIL（Synchronized Multimedia Integration Language）是 SVG 内置的动画标签系统。
它最大的优势是**不依赖外部样式和脚本**。如果你需要把 SVG 作为一张静态图片（例如用 `<img src="icon.svg">` 嵌入）并且还能动，那么 SMIL 是你唯一的选择！

<Callout type="warn" title="SMIL 的命运波折">
  Chrome 曾在 2015 年宣布要废弃 SMIL，这引发了社区的强烈抗议。后来因为 Web Animations API (WAAPI) 迟迟未能覆盖 SMIL 的所有功能（尤其是路径运动），Chrome 在 2016 年撤回了废弃决定。**现在你可以放心使用它了！**
</Callout>

## 核心元素介绍

### 1. 属性与变换动画 (`<animate>` & `<animateTransform>`)

- **`<animate>`**：用于在两个或多个属性值之间渐变（例如 `fill` 颜色、`r` 半径等）。
- **`<animateTransform>`**：专门用于控制元素的旋转、平移、缩放等变换操作。

### 2. 杀手级功能：路径运动 (`<animateMotion>`)

让元素沿着任意 `<path>` 路径运动，这是纯 CSS 很难做到的。通过 `<mpath>`，动画元素可以完美地沿着定义好的轨迹路径循环运动，甚至可以根据路径方向自动旋转！
