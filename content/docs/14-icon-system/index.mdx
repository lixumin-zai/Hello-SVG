在现代 Web 开发中，传统的 Iconfont (字体图标) 正在被全面淘汰，取而代之的是更加清晰、灵活、无障碍的 **SVG 图标系统**。
要构建一套好用的 SVG 图标库，第一步是规范图标的设计。

## 1. 统一的 ViewBox

不要让你的系统里混杂着 `viewBox="0 0 1024 1024"`、`viewBox="0 0 200 200"` 和 `viewBox="0 0 24 24"` 的图标。
**请强制所有图标使用相同的 viewBox（推荐 24x24 或 16x16）**。

统一 viewBox 的好处：
1. **默认大小一致**：不需要为每个图标单独写 CSS 微调大小。
2. **描边粗细一致**：在 `stroke-width="2"` 时，所有的图标看起来粗细是均匀的。

## 2. currentColor 的魔法

如果你把 SVG 的 `fill` 或 `stroke` 属性写死为 `#FF0000`，这个图标在暗色模式下可能就看不清了。
**currentColor** 是 CSS 中的一个关键字，它表示“当前元素继承的文本颜色”。把它用在 SVG 中，是图标自适应主题的终极秘诀！
