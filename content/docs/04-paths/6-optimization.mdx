# 4.6 路径工具与优化

手写简单的 `<path>` 命令（如矩形、简单的曲线）是有趣且有益的，但在实际项目中，我们很少手写复杂的路径数据。我们通常使用矢量图形软件生成它们，然后进行优化。

## 4.6.1 常用设计工具

以下工具可以导出高质量的 SVG 代码：

1.  **Adobe Illustrator**: 行业标准。导出时选择 "SVG" 格式，并注意设置 "Decimal Places"（小数位数）以控制精度。
2.  **Figma**: UI 设计首选。右键图层 -> "Copy as SVG"。
3.  **Sketch**: Mac 平台的经典设计工具。
4.  **Inkscape**: 免费开源的矢量绘图软件，功能强大，生成的 SVG 非常标准。

## 4.6.2 在线路径编辑器

如果你需要快速调整或生成路径，可以使用在线工具：

1.  **SVG Path Editor (yqnn.github.io)**: 非常直观的在线路径编辑器，可以直接拖拽控制点，查看对应的 d 属性。
2.  **Method Draw**: 在线 SVG 编辑器。

## 4.6.3 路径优化 (SVGO)

设计软件导出的 SVG 往往包含大量的冗余信息（如元数据、无用的组、过高的精度）。**SVGO (SVG Optimizer)** 是最流行的 SVG 优化工具。

### 优化策略

1.  **降低精度**：将 `10.1234567` 简化为 `10.12`。这能显著减少文件大小，且肉眼难以察觉差异。
2.  **使用相对坐标**：相对坐标通常比绝对坐标更短（数字更小）。
3.  **合并路径**：将多个相同样式的路径合并为一个 `<path>`。
4.  **移除无用数据**：删除编辑器元数据、隐藏的图层等。

### 示例

**优化前**:
```xml
<path d="M 10.000 10.000 L 50.000 50.000 L 20.000 50.000 Z" />
```

**优化后**:
```xml
<path d="M10 10l40 40-30 0z"/>
```
(去除了空格，使用了相对坐标，去除了不必要的小数点)

## 4.6.4 总结

`<path>` 是 SVG 的核心。理解它的工作原理能让你：
1.  看懂并手动微调 SVG 代码。
2.  利用 JS 动态生成路径，制作复杂的图表。
3.  制作路径动画（如描边动画、路径变形）。

虽然工具能帮我们生成代码，但"懂代码"能让我们更好地驾驭工具。
