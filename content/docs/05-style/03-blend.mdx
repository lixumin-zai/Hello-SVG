## 核心概念

### 1. 混合模式 (`mix-blend-mode`)

混合模式定义了一个元素的内容应该如何与它“下面”的内容（即背景或其他早先绘制的元素）混合。它不仅可以用于 SVG 内部的元素之间，**还可以让 SVG 元素与页面上的 HTML 背景进行混合**。

<Cards>
  <Card title="multiply (正片叠底)" icon="🌑">
    结果总是比原来的颜色更暗。就像把两张半透明的彩色塑料片叠在一起看。**常用于去除白色背景、模拟自然阴影。**
  </Card>
  <Card title="screen (滤色)" icon="☀️">
    结果总是比原来的颜色更亮。就像两台投影仪把不同的画面打在同一块屏幕上。**常用于去除黑色背景、模拟发光特效。**
  </Card>
  <Card title="overlay (叠加)" icon="🌗">
    结合了 multiply 和 screen，根据底色的明暗决定是加深还是提亮，从而**增加整体对比度**。
  </Card>
  <Card title="difference (差值)" icon="☯️">
    非常适合做**反色游标/反色文字**。当底色是白色时变黑，底色是黑色时变白，创造强烈的视觉反差。
  </Card>
</Cards>

### 2. 隔离上下文 (`isolation: isolate`)

这是一个常常被忽略但极其重要的 CSS 属性。

在上面的演示器中，如果你开启了 **isolation**，你会发现红、绿、蓝三个圆圈彼此之间仍然会混合（比如红色和绿色相交处变黄），**但它们不再与底部的背景图混合了**。

**工作原理**：
`isolation: isolate` 会创建一个新的“堆叠上下文（Stacking Context）”。
系统会先在这个隔离的容器（比如 `<g>` 标签）内部完成所有的混合计算，最终生成一张临时的、扁平的图像，然后再把这张临时图像以 `normal` 模式贴到大背景上。

<Callout type="tip" title="何时使用 isolation？">
  当你设计了一个由多个互相混合的形状组成的复杂 Logo 或组件，你不希望它放到用户的网页上时，因为受到用户网页深色/浅色背景的影响而改变原本的颜色，这时就必须给最外层的 <code>&lt;g&gt;</code> 加上 <code>isolation: isolate</code>。
</Callout>
