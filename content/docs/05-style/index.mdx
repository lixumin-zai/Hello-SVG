到目前为止，我们画的图形大多是黑白的。现在，是时候给它们上色了。SVG 的样式系统非常强大，既保留了 XML 的属性特性，又完美融合了 CSS 的灵活性。

## 本章概览

1.  **样式方法论**：属性 vs. CSS，哪种更好？
2.  **填充（Fill）**：不仅仅是颜色，还有透明度和规则。
3.  **描边（Stroke）**：线宽、端点、转角、虚线……描边的学问很大。
4.  **透明度与混合**：理解 `opacity` 的层叠效应。

## 5.1 SVG 样式的三种方式

给 SVG 元素上色有三种主要方式：

### 1. 表现属性 (Presentation Attributes)

这是最“SVG”的方式，直接写在标签上。

```xml
<circle fill="red" stroke="blue" stroke-width="5" />
```

-   **优点**：优先级最低，容易被 CSS 覆盖。语义清晰。
-   **缺点**：样式与结构耦合。

### 2. 内联样式 (Inline Style)

就像 HTML 一样。

```xml
<circle style="fill: red; stroke: blue;" />
```

-   **优点**：优先级高。
-   **缺点**：难以维护。

### 3. CSS (Cascading Style Sheets)

这是现代 Web 开发最推荐的方式。

```css
/* style.css */
.my-circle {
  fill: red;
  transition: fill 0.3s;
}

.my-circle:hover {
  fill: gold;
}
```

```xml
<circle class="my-circle" />
```

-   **优点**：样式分离，支持 `:hover` 等伪类，支持 CSS 动画。
-   **注意**：不是所有 CSS 属性都能用在 SVG 上（比如 `background-color` 无效，必须用 `fill`）。

## 推荐策略

1.  **布局与定位**（如 `x`, `y`, `width`, `height`）：使用**属性**。
2.  **外观与颜色**（如 `fill`, `stroke`）：使用 **CSS 类**。
3.  **动态交互**：使用 **JS 操作 CSS 类**或**内联样式**。
