画出了形状和路径后，我们需要给它们上色。SVG 提供了极其丰富的样式控制能力。

## 添加样式的三种方式

SVG 是一个神奇的规范，它允许你用三种不同的方式为同一个属性赋值（比如填充颜色）。它们的优先级从低到高如下：

<Tabs items={['表现属性 (最低)', 'CSS 样式表 (中等)', '内联 Style (最高)']}>
  <Tab value="表现属性 (最低)">
    **表现属性 (Presentation Attributes)** 是最常见、最基础的方式。直接将属性作为标签的 attribute 写上。
    
    ```xml
    <!-- 优先级最低，极易被外部 CSS 覆盖 -->
    <rect width="100" height="100" fill="red" stroke="black" stroke-width="2" />
    ```
    
    优势：结构清晰，适合作为**默认样式**。
  </Tab>
  <Tab value="CSS 样式表 (中等)">
    **内部/外部 CSS 样式表** 允许你像给 HTML 元素加样式一样给 SVG 元素上色。
    
    ```html
    <style>
      .my-rect { 
        fill: blue; 
        stroke: gray;
      }
    </style>
    
    <!-- 这里的 fill="red" 会被上面的 class="my-rect" 覆盖，最终显示蓝色 -->
    <rect class="my-rect" width="100" height="100" fill="red" />
    ```
    
    优势：批量管理样式，支持伪类（如 `:hover`）。
  </Tab>
  <Tab value="内联 Style (最高)">
    **内联 style 属性** 拥有最高的优先级，会覆盖表现属性和普通的 CSS 规则（除非 CSS 使用了 `!important`）。
    
    ```xml
    <!-- 最终渲染为绿色，因为 style 的优先级最高 -->
    <rect class="my-rect" fill="red" style="fill: green;" width="100" height="100" />
    ```
    
    优势：用于 JavaScript 动态计算出的、必须立刻生效的样式。
  </Tab>
</Tabs>

<Callout type="tip" title="最佳实践：为什么推荐表现属性？">
  在现代前端开发（特别是配合 Tailwind CSS 或 styled-components）中，强烈推荐使用**表现属性**（如 <code>fill="currentColor"</code>）来设置默认样式。因为它的优先级最低，你可以随时在外部用 CSS 类（如 <code>text-blue-500</code>）轻松覆盖它，而不用担心优先级冲突（Specificity Issues）。
</Callout>

## 核心概念解析

### 1. `fill` (填充色)
决定了图形内部的颜色。可以接受的值与 CSS 完全一致：
- 关键词：`red`, `blue`, `transparent`, `none`
- 十六进制：`#FF0000`
- RGB/HSL/OKLCH 等：`rgba(255, 0, 0, 0.5)`
- 甚至可以是指向渐变或图案的链接：`url(#myGradient)`（我们将在第七章详细学习）。

### 2. `none` 与透明的区别
在 SVG 中，`fill="none"` 和 `fill="transparent"`（或 `opacity="0"`）在视觉上看起来一样（都是透明的），但**在交互上有致命的区别**：
*   **`none`**：这个图形的内部是“空”的。你的鼠标点击会直接穿透它，点到它下面的元素。
*   **`transparent`**：图形内部有颜色，只是颜色是透明的。它**可以**接收鼠标的 `hover` 和 `click` 事件！

### 3. 神奇的 `currentColor`
当你把 SVG 的 `fill` 或 `stroke` 设置为 `currentColor` 时，它会**自动继承其父级 HTML 元素的文字颜色（color 属性）**。

这是构建现代 SVG 图标库（Icon System）的绝对核心技术！有了它，你在 React/Vue 中使用图标时，只需要给外层的 `<div>` 设置文字颜色，内部的 SVG 就会自动跟着变色，再也不需要通过 props 传颜色进去了。

```tsx title="React 中使用 currentColor 的典型模式"
// Icon.tsx
export const HeartIcon = ({ className }) => (
  <svg 
    viewBox="0 0 24 24" 
    // fill 继承父级颜色，或者由 className（如 text-red-500）覆盖
    fill="currentColor" 
    className={className}
  >
    <path d="M12 21.35l-1.45-1.32C5.4 15.36 2 12.28 2 8.5 2 5.42 4.42 3 7.5 3c1.74 0 3.41.81 4.5 2.09C13.09 3.81 14.76 3 16.5 3 19.58 3 22 5.42 22 8.5c0 3.78-3.4 6.86-8.55 11.54L12 21.35z" />
  </svg>
);

// App.tsx
// 只需要控制外层文字颜色（这里使用 Tailwind），图标自动变红！
<div className="text-red-500 hover:text-red-700 transition-colors">
  <HeartIcon className="w-6 h-6" />
</div>
```

### 4. 填充规则 (`fill-rule`) 的奥秘

当你绘制了一个**复杂多边形**（如五角星），或者一个包含多个子路径的**复合路径**（如通过 `M` 指令画了两个同心圆）时，它们往往会发生“自我交叉”。

此时，SVG 如何判断哪里是“里面”（需要填充），哪里是“外面”（不需要填充）？这就需要用到 `fill-rule` 属性。

<FillRuleVisualizer />

- **`nonzero` (非零环绕)**：默认值。它的计算方法是：从该区域向外射出一条射线。如果路径是**顺时针**穿过射线，计数器加 1；如果是**逆时针**穿过，计数器减 1。最后如果计数器**不为 0**，就认为这里是“内部”，进行填充。
- **`evenodd` (奇偶环绕)**：更容易理解的模式。同样射出一条射线，只要它与路径的交点总数是**奇数**，就是内部；是**偶数**，就是外部。

<Callout type="tip" title="实战技巧：如何用默认的 nonzero 画出空心圆环？">
  在上面的演示中，同向（比如都是顺时针）绘制的同心圆在 `nonzero` 规则下，中心会被填满。
  但在实际开发中，如果我们不想修改 `fill-rule`，只需**将内圆的绘制方向改为逆时针**（顺时针+1，逆时针-1，抵消为 0），中心就会变成“外部”，从而实现完美的镂空效果！
</Callout>
